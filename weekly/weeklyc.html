Aaaaaaanyways, these past few weeks I've been messing with adding JavaScript into my code (usually in a separate file, but sometimes I just use the script tag.). I had initially thought that JavaScript was pretty much the same thing as Java, but I was pretty far off. Still, it hasn't been too difficult to learn - especially with the help of online resources.

I attempted to utilize JavaScript for our first big assignment, with mixed results. HTML's lack of variables and ability to do much on its own (especially without a server) means that JS is doing some of the heavier lifting in terms of coding.

It's pretty cool to see three different programming languages tackling three different areas all at the same time: HTML helps with layout, positioning, and linking, CSS helps with styling, and JS allows for greater user interactivity.

It's also fun to see how the programming languages interact with each other, and how they perform the same operations in different ways. For instance, Linking to the top of a page in HTML is as simple as adding an < a > tag and href to another part of the page with a specific ID or class.

Javascript is pretty similar; it uses "document.getElementById("ID here")", where you can put the same ID into the parantheses. This allows the code to focus in on the specific part with that ID; then, if one wishes to scroll there, they can add ".scrollIntoView()" to the end of the code to scroll.

I'm sure you can also do other things that require referencing specific elements by ID (maybe changing color, adding new effects, etc.). I can already imagine the uses one might have for this in connection to CSS.
